What is GitHub?

GitHub is a web-based platform for version control and collaboration, built on top of Git.

It allows developers to:

  • Host and manage Git repositories
  • Track changes to code
  • Collaborate on projects via branches and pull requests
  • Review code and manage issues
  • Automate workflows with GitHub Actions

Repositories (Repos)

A repository is a project folder containing all files and revision history.

git init             # Initialize a new repo
git clone URL         # Clone an existing repo

Repos can be public (open source) or private.

Branching

Branches let you work on different features or fixes independently.

git branch feature/login      # Create a new branch
git checkout feature/login     # Switch to the branch
git merge feature/login        # Merge changes back into main

Common branch strategies: main/master, develop, feature branches.

Pull Requests (PRs)

PRs are proposed changes submitted for review before merging into a branch.

  • Discuss and review code
  • Run automated tests
  • Merge approved changes

Collaboration

  • Forking: Copy a repo to your account to experiment without affecting the original.
  • Issues: Track bugs, tasks, and feature requests.
  • Code Review: Comment on PRs to improve code quality.
  • GitHub Actions: Automate builds, tests, and deployments.

Best Practices

✅ Write clear commit messages.
✅ Keep branches focused on one task.
✅ Review PRs thoroughly before merging.
✅ Use .gitignore to exclude unnecessary files.
✅ Protect main branch with required reviews and tests.
✅ Document repo with README and CONTRIBUTING files.
✅ Use tags/releases for versioning.
✅ Regularly sync forked repos to stay updated.
