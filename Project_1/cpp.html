What is C++?

C++ is a powerful, high‑performance, general‑purpose programming language developed as an extension of C.

Widely used for:

  • System/software development
  • Game development
  • Real‑time simulations
  • High‑performance applications
  • Embedded systems

C++ Basics

Hello World

#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

Data Types

  • int, float, double, char, bool
  • string (from <string>)
  • User‑defined types with struct and class

Basic Syntax

  • Statements end with ;
  • Curly braces { } define blocks
  • Single‑line comments: // comment
  • Multi‑line comments: /* comment */

Control Flow

int x = 10;

if (x > 0) {
    std::cout << "Positive" << std::endl;
} else {
    std::cout << "Non‑positive" << std::endl;
}

for (int i = 0; i < 5; ++i) {
    std::cout << i << std::endl;
}

int count = 0;
while (count < 5) {
    std::cout << count << std::endl;
    ++count;
}

Functions

int add(int a, int b) {
    return a + b;
}

Object‑Oriented Programming (OOP)

Classes and Objects

class Person {
public:
    std::string name;

    Person(std::string n) : name(n) {}

    void greet() {
        std::cout << "Hello, " << name << std::endl;
    }
};

int main() {
    Person p("Alice");
    p.greet();
    return 0;
}

Key OOP Concepts

Encapsulation

Data hiding using private and protected access specifiers.

Inheritance 
class Animal {
public:
    void speak() {
        std::cout << "Animal sound" << std::endl;
    }
};

class Dog : public Animal {
public:
    void speak() {
        std::cout << "Woof!" << std::endl;
    }
};

int main() {
    Dog d;
    d.speak();
    return 0;
}
Polymorphism (Virtual Functions) 
class Animal {
public:
    virtual void speak() {
        std::cout << "Animal sound" << std::endl;
    }
};

class Cat : public Animal {
public:
    void speak() override {
        std::cout << "Meow" << std::endl;
    }
};

void makeSpeak(Animal* a) {
    a‑>speak();
}

int main() {
    Animal* a = new Cat();
    makeSpeak(a);
    delete a;
    return 0;
}
Constructors and Destructors 
class Resource {
public:
    Resource() {
        std::cout << "Acquired resource" << std::endl;
    }
    ~Resource() {
        std::cout << "Released resource" << std::endl;
    }
};

Standard Template Library (STL)

Powerful library of generic data structures and algorithms.

Common Containers

  • std::vector — dynamic array
  • std::list — doubly linked list
  • std::map — sorted key‑value map
  • std::unordered_map — hash table
  • std::set — sorted unique elements

Example: Vector Usage

#include <vector>
#include <iostream>

int main() {
    std::vector<int> numbers = {1, 2, 3, 4, 5};
    numbers.push_back(6);

    for (int n : numbers) {
        std::cout << n << " ";
    }
    std::cout << std::endl;
    return 0;
}

Algorithms

Functions like std::sort, std::find, and many others.

#include <algorithm>  // for std::sort
#include <vector>

std::vector<int> v = {3, 1, 4, 1, 5};
std::sort(v.begin(), v.end());

Advanced Concepts

Templates (Generic Programming) 
template <typename T>
T add(T a, T b) {
    return a + b;
}

int main() {
    std::cout << add(3, 4) << std::endl;
    std::cout << add(3.5, 2.5) << std::endl;
    return 0;
}
Exception Handling 
try {
    // code that may throw
} catch (const std::exception& e) {
    std::cerr << "Error: " << e.what() << std::endl;
}
Smart Pointers 
#include <memory>

std::unique_ptr<int> p1(new int(5));  // automatically deletes when out of scope
std::shared_ptr<int> p2 = std::make_shared<int>(10);
Move Semantics & Rvalue References

Optimize performance by avoiding unnecessary copies.

void foo(std::string&& s) {
    std::cout << s << std::endl;
}

Best Practices

✅ Use RAII (Resource Acquisition Is Initialization) for resource management.
✅ Prefer STL containers over raw arrays.
✅ Avoid raw pointers; use smart pointers where possible.
✅ Follow consistent naming conventions.
✅ Write small, reusable functions and classes.
✅ Use const‑correctness.
✅ Handle exceptions appropriately.
✅ Comment and document your code clearly.
