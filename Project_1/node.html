What is Node.js?

Node.js is a JavaScript runtime built on Chrome's V8 engine that allows you to run JavaScript on the server.

Key features:

  • Event-driven, non-blocking I/O model
  • Built for scalable network applications
  • Single-threaded with asynchronous callbacks
  • Vast ecosystem via npm

Modules

Node.js uses CommonJS modules. Each file is a module with its own scope.

// Exporting a function
module.exports = function greet() {
  console.log("Hello, Node.js!");
};

// Importing a module
const greet = require('./greet');
greet();

In newer versions, ES Modules (import/export) syntax is also supported.

Event Loop & Asynchronous Programming

Node.js uses an event loop to handle asynchronous operations without blocking the main thread.

// Example: Non-blocking file read
const fs = require('fs');

fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

console.log('Reading file asynchronously');

The callback runs after the file is read, allowing other code to run meanwhile.

Core Modules

  • fs - File system operations
  • http - Create HTTP server/client
  • path - Work with file and directory paths
  • events - Event emitter API
  • os - Operating system info
  • crypto - Cryptography

npm & Packages

npm (Node Package Manager) is the default package manager for Node.js.

// Install express (web framework)
npm install express

// Add to package.json
{
  "dependencies": {
    "express": "^4.18.2"
  }
}

You can use npm scripts for automation, and publish your own packages.

Best Practices

✅ Use asynchronous APIs to avoid blocking.
✅ Handle errors properly in callbacks and promises.
✅ Modularize your code with small reusable modules.
✅ Use environment variables for configuration.
✅ Keep dependencies up-to-date and audit for vulnerabilities.
✅ Write tests for critical parts of your app.
✅ Use linting tools like ESLint for consistent code style.
✅ Use process managers (PM2) for production deployments.
